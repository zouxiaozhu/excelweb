搜索结果处理验证

数据处理流程

📊 数据流转过程：

  1. API原始响应：服务器返回完整的响应对象
  2. 响应拦截器处理：当 code === 0 时，返回 data.data
  3. 组件接收：直接接收到处理后的数组数据
  4. 数据转换：将 columnData 转换为表格行数据
  5. 表格显示：使用 tableColumns 显示数据

API原始响应格式

服务器返回的完整响应：

{ "success": true, "code": 0, "message": "ok", "data": [ { "rowLine": 2, "columnData": { "所在位置": { "id": 13650, "columnName": "所在位置", "columnValue": "所在位置(示例)" }, "生产厂家": { "id": 13653, "columnName": "生产厂家", "columnValue": "生产厂家(示例)" } // ... 其他列数据 } } ] }

响应拦截器处理后

组件实际接收到的数据（response）：

[ { "rowLine": 2, "columnData": { "所在位置": { "id": 13650, "columnName": "所在位置", "columnValue": "所在位置(示例)" }, "生产厂家": { "id": 13653, "columnName": "生产厂家", "columnValue": "生产厂家(示例)" } // ... 其他列数据 } } ]

数据转换过程

代码中的转换逻辑：

// 转换数据格式以适配表格显示 searchResults.value = response.map((item: any) => { const rowData: any = {} Object.keys(item.columnData).forEach(columnKey => { rowData[columnKey] = item.columnData[columnKey].columnValue }) return rowData })

可能的问题排查

问题：搜索结果为空

排查：检查API响应是否为数组，确认columnData结构

问题：表格列不显示

排查：确认tableColumns是否正确生成，检查prop与columnData键名匹配

问题：数据错位

排查：确认columnData中的字段名与表格列的prop完全一致

问题：转换失败

排查：检查item.columnData是否存在，确认columnValue字段
